# 大型央企集团穿透式监管平台（一平台三中心）5 年落地实操方案

**实施周期**：2026 年 9 月 1 日 - 2031 年 8 月 31 日

**适配前提**：复用现有浪潮一体化管控平台（海岳 iGIX 开放架构）、集团司库系统（MySQL/Oracle 双库支撑），全链路兼容国产信创环境（鲲鹏 / 海光芯片、银河麒麟 / 统信 OS、达梦 / 人大金仓 DB），云原生容器化部署，所有技术组件为 GitHub 高星、Apache2.0/MIT 协议开源项目，无商业版权风险

**终极量化目标**：支撑 1000 人同时在线，核心页面响应≤2s，数据采集准确率≥99.9%，风险识别命中率≥85%，全链路业务可用性≥99.95%

### 2.1 数据采集中心选型理由与候选对比

**Airbyte（21.6k★，MIT/ELv2）— 多源异构 API 采集首选**
600+ 官方连接器覆盖 SaaS、数据库、数据湖、向量库，自 2025 年 v2.0 起内置 Debezium 作为 CDC 库，支持日志级实时增量同步；容器化隔离执行、Connector Development Kit（CDK）可在 30 分钟内自研浪潮/司库定制连接器。对比 Fivetran（商业闭源、按行计费）、StreamSets（IBM 商业、无开源版），Airbyte 是唯一兼具高星开源、海量连接器、信创可私有化部署的方案。

**DataX（12.5k★，Apache 2.0）— 离线全量批量抽取**
阿里开源异构数据源同步框架，插件化架构支持 MySQL/Oracle/达梦/人大金仓等关系库及文件系统，单机并发可达万级条/s，适合浪潮 3 年历史业务数据回灌 ODS 层。与 Sqoop（已废弃、社区停滞）相比，DataX 流式读写、内存占用更低，国产数据库适配更成熟。

**Debezium（12.0k★，Apache 2.0）— 实时 CDC 增量采集**
基于数据库事务日志（MySQL binlog / Oracle redo / PostgreSQL WAL）的无侵入变更捕获，对接 Kafka Connect 生态，毫秒级延迟保障司库 16 张资金交易表实时同步。对比 Canal（仅支持 MySQL、社区活跃度下降）、Flink CDC（封装 Debezium、依赖 Flink 集群、运维成本更高），独立部署的 Debezium 更轻量、对 Oracle 支持更完整，契合司库 Oracle 双库场景。

**Apache Tika（2.0k★，Apache 2.0）— 非结构化文档解析**
支持 PDF/Word/Excel/音视频等 1000+ 格式元数据与正文提取，是 Apache 旗下成熟的内容检测框架。合同附件、凭证扫描件、招标文档经 Tika 解析后入库，供阶段 4 大模型做违规条款审查。

**Apache DolphinScheduler（14.1k★，Apache 2.0）— 采集任务编排**
去中心化多 Master/Worker 架构，原生支持水平扩容，内置 30+ 任务类型（Shell/SQL/Spark/Flink/HTTP/DataX/Python），可视化拖拽 DAG，注册中心可选 JDBC/ZK/Etcd。对比 Airflow（37k★，Apache 2.0）：Airflow 需写 Python DAG、依赖 Celery+Redis/RabbitMQ、Scheduler 单点、运维重；DolphinScheduler 低代码拖拽、数据分析师可直接上手、JDBC 注册中心复用业务库免额外部署 ZK，更匹配央企数据团队人员结构。

### 2.2 智慧监督中心选型理由与候选对比

**Apache Doris（18.6k★，Apache 2.0）— 实时数据仓库（首选）**
MPP 架构 + CBO 优化器，自动选择 Broadcast/Shuffle/Colocate Join，多表关联比 ClickHouse 快 2-10 倍；Merge-on-Write 引擎支持高频 UPSERT、强一致实时可见，千级并发查询；MySQL 协议兼容、标准 SQL，运维门槛低；3.0+ 存算分离开源、弹性扩缩容免数据重平衡。快手、腾讯音乐、有赞等已从 ClickHouse 迁移至 Doris（核心查询快 2-3 倍、复杂 JOIN 快 2-10 倍、可跑通 ClickHouse OOM 的查询）。对比 ClickHouse（~48k★）：单表聚合极致、但多表 JOIN 弱（约 50% TPC-DS 查询失败）、并发通常 <100、更新异步最终一致、存算分离仅商业云版。穿透监管场景以多表关联下钻、高并发即席查询为主，Doris 显著优于 ClickHouse。

**NebulaGraph（11k+★，Apache 2.0）— 关联关系图谱（首选）**
Shared-nothing 原生分布式图数据库，Raft 协议保证强一致，存算分离线性扩容，支撑千亿顶点万亿边、毫秒级查询；nGQL 兼容 OpenCypher，DB-Engines 国内图数据库排名第一，京东、美团、小红书生产验证。资金流向、股权控制、隐性关联交易穿透需多跳长路径遍历，NebulaGraph 分布式遍历性能领先。对比 Apache HugeGraph（百度开源，TinkerPop3/Gremlin+Cypher 双语法、全栈含图计算/GraphRAG）、Neo4j（GPLv3 商业受限、分布式为商业版功能、社区版单机）：Neo4j 协议存在商业风险且横向扩展受限；HugeGraph 全栈能力强但分布式成熟度与生产案例不及 NebulaGraph。NebulaGraph 在央企信创、超大规模图谱场景综合最优。

**Drools（8.5k★，Apache 2.0）— 合规规则配置**
完整 RETE 算法实现，KIE 工作台支持规则可视化编辑、版本管控，适合上百条复杂金融风控规则的集中管理。对比 LiteFlow（dromara 开源，3.8k★ GitHub/7.5k★ Gitee，Apache 2.0）：LiteFlow 擅长组件化业务编排、规则热刷新、支持 Groovy/Python/Aviator 多脚本节点、AI Agent 编排能力强，但复杂规则网络推理能力弱于 Drools。本方案采取"Drools 主规则推理 + LiteFlow 编排核查工单联动"的组合策略，兼顾规则深度与流程灵活性。

**LangChain（100k+★，MIT）+ Dify（135k★，Apache 2.0）— 智能分析**
LangChain 提供大模型应用编排底座，LangGraph（31.2k★）状态图能力支撑多步推理；Dify 提供全栈 AI 平台（RAG/Agent/工作流/模型管理可视化），降低非算法人员配置智能应用的门槛。自然语言穿透查询、合同违规审查、风险报告自动生成基于 LangChain + 微调 Llama3 构建，Dify 作为可视化编排与运维入口。

**Apache Flink（25.4k★，Apache 2.0）— 流计算风险匹配**
原生流处理、毫秒级延迟、精确一次语义，适合资金流水实时规则匹配与窗口聚合。对比 Apache Spark（42.2k★）Structured Streaming 微批模式（秒级延迟），Flink 在低延迟实时风控场景更优。

### 2.3 调度指挥中心选型理由与候选对比

**Flowable（7k+★，Apache 2.0）— 核查工单流程引擎（首选）**
源自 Activiti 原班人马，同时支持 BPMN 2.0 + CMMN（案例管理）+ DMN（决策表）三开放标准，社区活跃度高于 Activiti、GitHub 星标高于 Camunda，Spring Boot 集成成熟。"风险告警→工单分派→线下核查→结果录入→整改复核→归档结案"标准流程基于 BPMN，复杂核查案例用 CMMN，规则分派用 DMN。对比 Camunda（~4k★，Apache 2.0）：Camunda 社区版自带 Cockpit/Tasklist/Admin 监控界面、企业级监控更强，但 Camunda 8 转向云原生订阅、社区版功能收敛；Activiti 7 维护趋缓。Flowable 三引擎一体、纯 Apache 2.0、央企自主持久可控，为综合最优。

**Apache Superset（73k+★，Apache 2.0）— 可视化指挥大屏（首选）**
Apache 基金会最活跃的数据可视化项目，40+ 图表类型 + 插件架构，SQL Lab IDE + Jinja 模板 + 虚拟数据集，原生对接 50+ 数据库（含 Doris/ClickHouse/Trino），RBAC 权限 + 行级数据隔离，Dropbox/Spotify/Lyft 生产使用。对比 Metabase（48.2k★，AGPL-3.0 CE）：Metabase 无代码体验优、适合非技术用户，但社区版 AGPL 协议有传染性风险、图表类型与 SQL 灵活度不及 Superset；Grafana（62k+★，AGPL-3.0）专精时序运维监控、非业务 BI。指挥驾驶舱需复杂下钻、多源联邦、高度定制可视化，Superset 纯 Apache 2.0 + SQL 原生灵活度最高。

**Apache RocketMQ（24.3k★，Apache 2.0）— 三中心异步解耦**
阿里开源金融级消息中间体，事务消息、严格顺序、亿级消息堆积能力，契合资金交易高可靠异步解耦与削峰。对比 Kafka（18.8k★，单机吞吐高但事务与延时消息弱）、Pulsar（12.4k★，存算分离、多云友好但运维复杂），RocketMQ 在金融交易级可靠性、事务消息、定时/延时消息上最贴合监管工单与资金告警场景。

**Apache HertzBeat（7.3k★，Apache 2.0）— 全链路监控告警**
无代理（agentless）下一代实时可观测系统，支持应用/数据库/中间件/云原生多类指标统一采集与阈值告警，集成 Prometheus 生态。配合 Prometheus（65k★）+ Grafana（62k+★）构建"业务告警（HertzBeat）+ 指标采集（Prometheus）+ 运维大盘（Grafana）"分层监控，Grafana 仅限内部运维、不分发以规避 AGPL。

### 2.4 技术中台选型理由与候选对比

**Apache APISIX（15k+★，Apache 2.0）— 统一流量网关（首选）**
NGINX + LuaJIT + etcd 架构，无状态数据面、毫秒级配置热更新（Kong 依赖 PostgreSQL 轮询有延迟），单核 23000 QPS、0.2ms 延迟，100+ 全开源插件（OIDC/限流/可观测/AI 网关），HTTP/1-3/gRPC/WebSocket/MQTT/Dubbo 全协议。对比 Kong（~42.5k★，Apache 2.0 + Enterprise）：Kong 高级特性（OIDC、OPA、mTLS、FIPS）锁在企业版、企业版起步约 $5350/年；APISIX 全部能力开源、零供应商锁定、Apache 基金会治理。央企追求自主可控、无商业付费墙，APISIX 综合最优（Severstal 等企业 2026 年 OIDC+Keycloak+OPA 实测验证）。

**Keycloak（34.8k★，Apache 2.0）— 统一身份细粒度认证**
Red Hat 开源 IAM，支持 OIDC/OAuth2/SAML/LDAP，对接集团 SSO 实现单点登录，行级数据权限与多租户隔离成熟。对比 Casdoor（13.7k★，Apache 2.0，Go 语言 + Casbin）：Casdoor 轻量、API 友好，但企业级 SSO 协议矩阵与生态集成不及 Keycloak；Keycloak 与 APISIX 的 openid-connect 插件原生联动，构成统一鉴权链路。

**Apache Ranger（3.8k★，Apache 2.0）+ Prometheus（65k★）+ Grafana（62k+★）**
Ranger 提供数据安全脱敏与细粒度访问控制（库表列行级），Prometheus + Grafana 构成运维技术与业务指标监控基线。

### 2.5 前端展示层选型（匹配现有监管总览工程）

前端栈严格对齐已落地的新兴际华穿透式监管平台·监管总览工程实际代码栈（React 18 + TypeScript + Vite 6 + Tailwind CSS 3.4 + Recharts 2.15 + Zustand + React Router 7 + Lucide React），保障总览/穿透查询/风险预警/关系图谱/指挥大屏/财务监管/核查工单七大页面与后端三中心无缝衔接：

- **React 18 + TypeScript**：并发渲染 + 类型安全，支撑 1000 人并发下 ≤2s 响应；
- **Vite 6（70k+★，MIT）**：ESM 原生构建、毫秒级 HMR，替代 Webpack 提升研发效能；
- **Tailwind CSS（84k+★，MIT）**：原子化 UI + 暗色/亮色主题 Token（远立蓝主色），保障监管大屏与日常办公双视觉一致性；
- **Recharts（27k+★，MIT）**：声明式 React 图表库，支撑健康度柱状、风险环形、风险趋势等监管可视化组件；
- **Zustand（50k+★，MIT）**：轻量状态管理（themeStore/layoutStore），无 Redux 样板代码，降低维护成本；
- **React Router 7**：嵌套路由 + 数据加载，支撑多级穿透下钻导航。

全栈 MIT 协议、无传染性风险，与后端 Apache 2.0 组件共同构成完整自主可控的开源技术体系。

## 三、分阶段详细落地方案

### 阶段 1：基础底座搭建与核心系统直连（2026.09-2027.08）

**阶段目标**：打通浪潮一体化管控平台、集团司库系统的标准化数据采集链路，完成数据采集中心高可用集群部署，破除首层级数据孤岛，实现业务数据的稳定、标准化采集。

#### 1.1 核心落地任务

##### （1）基础集群部署



* 搭建 Production 级 K8s 集群：3 台 Master 节点、4 台 Worker 节点，部署 Rook-Ceph 分布式存储、Harbor 私有镜像仓库、CoreDNS 服务发现，集群启用 RBAC 权限隔离，所有节点关闭 Swap 分区、优化内核参数；

* 部署数据采集中心核心组件：Airbyte v1.2、DataX v3.0、Debezium v2.4、Apache DolphinScheduler v3.2、Apache Atlas v2.6、Great Expectations v0.18，所有组件采用特定版本镜像，固定标签化部署，避免版本冲突。

##### （2）浪潮平台适配对接



* 通过 Airbyte 官方 REST API 连接器，对接浪潮 iGIX 平台开放的标准 OpenAPI 接口，申请业务系统数据读取权限，配置**32 张核心业务表**（组织架构、客商主数据、项目档案、合同头表、财务凭证主表）的每日增量采集同步任务，定时拉取业务系统新增及变更数据；

* 额外部署 DataX 浪潮定制适配器，配置离线采集任务，批量抽取近 3 年历史业务数据，经过格式转换、脏数据过滤后，落地数据仓库 ODS 原始数据层；

* 在浪潮平台侧配置 API 访问白名单、流控策略，限制单接口并发请求数，联合调试接口超时重试机制，保障采集过程不影响业务系统正常运行。

##### （3）司库系统适配对接



* 开启司库 MySQL 数据库 binlog（ROW 模式），Debezium 连接器以无侵入性方式解析 binlog 日志，实时增量采集**16 张资金交易表**（账户流水、支付订单、往来台账、担保记录）的增删改数据；

* 下游对接 RocketMQ v4.9 流量削峰集群，将实时采集的资金数据写入消息队列，再由消费端同步至数据仓库，避免高并发读取压垮业务数据库；

* 针对司库 Oracle 数据库的部分核心数据，额外配置 DataX 定时全量采集任务，每日低峰期抽取基础账户档案、编码字典类数据，保障基础数据一致性。

##### （4）元数据与数据治理落地



* 通过 Apache Atlas 逆向采集浪潮、司库业务库的表结构、字段注释、主外键关联，自动生成业务数据血缘关系图，梳理所有采集表的业务口径、责任人、使用场景；

## 四、重点业务场景落地架构

### 4.1 财务资金穿透监管场景

**场景需求**：实时追踪资金全链路交易记录，穿透识别违规资金占用、异常交易、虚假贸易、违规担保，支撑从集团汇总数据到末级交易凭证的全链路溯源。

**落地流程**：



1. **数据采集**：通过 Debezium 实时采集司库资金交易流水，通过 DataX 定时采集浪潮财务凭证、合同数据，通过 Airbyte 采集非结构化合同附件、银行回单；

2. **数据存储**：标准化数据落地 Apache Doris 数仓，资金关联关系落地 NebulaGraph 图数据库；

3. **风险识别**：drools 规则引擎匹配大额交易、关联交易等标准化风险，NebulaGraph 穿透挖掘隐性资金流向，AI 模型识别规则难以覆盖的隐性风险；

4. **告警处置**：风险数据写入 HertzBeat 触发告警，自动生成核查工单，流转至业务人员处置；

5. **穿透查询**：用户可从集团资金汇总数据，逐级下钻至二级单位、三级子公司明细数据，再到具体交易流水、财务凭证、合同附件。

### 4.2 投资决策穿透监管场景

**场景需求**：覆盖项目立项、招标采购、合同签订、资金支付、竣工转资全链路，识别超概算、违规分包、虚假招标、未批先建风险，支撑立项与执行数据多维度对比。

**落地流程**：



1. **数据采集**：通过 DataX 定时采集浪潮项目管理、采购、合同数据，通过 Debezium 实时采集司库投资类资金支付数据，通过 Airbyte 采集招标文档、批复文件、施工图纸等非结构化数据；

2. **数据存储**：标准化数据落地 Apache Doris 数仓，项目关联关系落地 NebulaGraph 图数据库；

3. **风险识别**：drools 规则引擎匹配项目超概算、违规分包、投标异常等标准化风险，AI 模型解析招标文档识别虚假招标，图数据库穿透识别隐性关联分包；

4. **告警处置**：风险数据写入 HertzBeat 触发告警，自动生成核查工单，流转至业务人员处置；

5. **穿透查询**：用户可从集团投资汇总数据，逐级下钻至项目概算、执行进度、资金支付明细，再到具体招标文档、合同附件、现场签证记录。

### 4.3 合规风控穿透监管场景

**场景需求**：覆盖全业务流程，识别违规担保、关联交易舞弊、虚假贸易、不合规发票，支撑多源数据交叉验证，满足内外部审计要求。

**落地流程**：



1. **数据采集**：通过 DataX 定时采集浪潮财务、合同、采购数据，通过 Debezium 实时采集司库资金交易数据，通过 Airbyte 采集合同附件、发票影像、审批扫描件等非结构化数据；

2. **数据存储**：标准化数据落地 Apache Doris 数仓，业务关联关系落地 NebulaGraph 图数据库；

3. **风险识别**：drools 规则引擎匹配交易异常、发票异常、合同异常等标准化风险，图数据库穿透挖掘隐性关联交易，AI 模型解析非结构化数据交叉验证业务真实性；

4. **告警处置**：风险数据写入 HertzBeat 触发告警，自动生成核查工单，流转至业务人员处置；

5. **穿透查询**：用户可从集团合规风险汇总数据，逐级下钻至风险明细、关联交易对手、资金流水、合同附件、审批记录，形成完整审计证据链。

### 4.4 安全生产穿透监管场景

**场景需求**：对接安全生产系统、智能设备监控平台，实时采集现场音视频、设备运行数据，识别违规操作、安全隐患、整改超时，支撑快速溯源，联动告警处置。

**落地流程**：



1. **数据采集**：通过 DataX 定时采集安全生产系统隐患排查、整改记录数据，通过 Airbyte 实时采集智能设备音视频、运行状态传感器数据，通过 Debezium 实时采集安全生产系统变更数据；

2. **数据存储**：标准化数据落地 Apache Doris 数仓，设备关联关系落地 NebulaGraph 图数据库；

3. **风险识别**：drools 规则引擎匹配设备状态异常、整改超时、区域闯入等标准化风险，AI 模型分析音视频数据识别人员未佩戴安全帽、违规操作；

4. **告警处置**：风险数据写入 HertzBeat 触发告警，联动现场语音对讲，自动生成核查工单流转至安全业务人员处置；

5. **穿透查询**：用户可从集团安全生产汇总数据，逐级下钻至区域、现场实时监控画面、设备运行状态、处置记录、人员资质信息。
